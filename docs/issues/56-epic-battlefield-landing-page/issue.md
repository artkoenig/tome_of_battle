## Description
# PRD: Epic Battlefield Landing Page

## Problem Statement / Bug Description
Currently, GitHub Pages serves the automated project status report at the root URL (`/`), leaving prospective users without an engaging, high-performance product landing page to learn about "Tome of Battle" and launch the app.

## Solution
Create a modern, visual, high-performance one-pager landing page styled in the "Epic Battlefield" dark fantasy aesthetic.
- The new landing page will be served at the root (`/index.html`) on GitHub Pages.
- All landing page copy, headings, and CTAs will be strictly in English.
- The existing automated status report will move to `/status` (`/status/index.html`), visually updated to fit the dark fantasy design system.
- The landing page header will include navigation (Features, How it Works, Status, "Launch App" CTA).
- Product screenshots in the showcase section will strictly feature mobile viewports.
- Architectural Decision Records (ADRs) will not be linked anywhere on the public landing page.

## User Stories / Requirements
1. As a tabletop gamer / prospective user, I want an epic dark fantasy landing page (English) so that I can quickly understand what "Tome of Battle" is and open the app.
2. As a mobile user, I want to see mobile screenshot previews of the army builder interface so that I know how it works on smartphones.
3. As a user or maintainer, I want a direct link to the Project Status (`/status`) in the header navigation so that I can inspect quality gates and test coverage.
4. As a user, I want clear feature highlights (Offline PWA, Battlescribe .cat support, instant solver/validator) and a 3-step Quickstart guide.

## Technical Decisions
- Affected Modules:
  - Landing Page static source and assets (`index.html`, responsive styles, mobile screenshot assets).
  - Project State Generator (`scripts/project-state/generate.js` & `renderReport.js`) updated to output to `/status/index.html` and match the dark fantasy theme.
  - GitHub Actions Workflow (`.github/workflows/status-report.yml`) updated to assemble both `/index.html` (Landing Page) and `/status/index.html` (Project Status) into `./_site`.
- Technical Clarifications / Architectural Decisions:
  - Landing page built as a clean, responsive static site with vanilla CSS/HTML/JS for maximum speed and compatibility on GitHub Pages.
  - Dark fantasy design system ("Epic Battlefield") using dark slate backgrounds, crimson/brass accents, and high-quality atmospheric background illustrations.
  - ADR pages (`/adr/*`) remain generated by Jekyll under `./_site/adr/`, but are intentionally unlinked from the public landing page UI.

## Testing Decisions
- Modules to Test:
  - Landing Page HTML structure and asset links.
  - Project State Report build output path (`./_site/status/index.html`).
  - Mobile screenshot assets and responsive viewport rendering.
- Test Interfaces (Seams):
  - GitHub Pages build script (`node scripts/project-state/generate.js`) verification.
  - Static HTML link & validation checks.

## Out of Scope
- Linking ADRs on the landing page or footer.
- Desktop viewport screenshots in the landing page showcase section.
- German language copy on the landing page (all landing page content is strictly English).

## Acceptance Criteria
- [ ] Landing page (`/`) is rendered in English with the "Epic Battlefield" dark fantasy theme.
- [ ] Navigation includes Features, How it Works, Status (`/status`), and "Launch App" CTA.
- [ ] Screenshot showcase exclusively features mobile screenshots.
- [ ] Automated status report is hosted under `/status` and matches the dark fantasy styling.
- [ ] No links to `/adr/` exist on the landing page or footer.
- [ ] Build workflow (`status-report.yml`) builds both landing page and status report into `./_site`.
